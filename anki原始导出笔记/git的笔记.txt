使用 git pull --rebase 拉取更新后，GitHub 上的版本会被改变吗？	不会。git pull --rebase 只拉取并在本地变基，只有执行 git push 才会更新 GitHub。

已确定 GitHub 是最新正确版本，本地分支有问题，想以远程为主，git fetch origin & git reset --hard origin/你的分支名？为什么使用这两个指令？	因为 fetch 会更新本地对远程分支的最新引用，确保 reset 基于的是 GitHub 最新版本。

Carriage Return 和 Line Feed Carriage 不是马车，feed 不是喂食吗？	在打字机术语里：    Carriage（字车）：指打字机上移动的承纸机构，并非“马车”。   Feed（送进）：指机械将纸张向下送一行的动作，不是“喂食”。
Carriage Return 为什么叫“回车”，Line Feed 为什么叫“换行”？	Carriage Return：来源于打字机，将“字车”拉回到行首，所以叫回车。   Line Feed：让纸向下走一行，所以叫换行。
回车（CR）和换行（LF）的英文全称是什么？	CR：Carriage Return   LF：Line Feed
“CRLF：回车+换行” 应该怎么理解？回车不是换行吗？	回车（CR）和换行（LF）是两个不同的控制符：   CR：把光标移到行首   LF：让光标向下移动一行   Windows 使用 CR+LF 两个字符来表示换行，Unix 系统只用 LF。
Windows 和 Unix 系统分别是用什么表示换行	Windows 使用 CR+LF 两个字符来表示换行，Unix 系统只用 LF。
Git 仓库通常存储为 LF，这是为什么？	因为 LF 是跨平台协作的通用标准，能避免不同系统的换行差异造成冲突。
在老式电传打字机上，回车（CR）和换行（LF）的效果分别是什么？	CR：让打印头回到行首   LF：让纸上滚一行
为什么主流使用 LF 而不是 CRLF？	因为 LF 是多数派：Unix/Linux/macOS 主流使用，服务器多跑在 Linux，上游开源项目也普遍采用 LF。
